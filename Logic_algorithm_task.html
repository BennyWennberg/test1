Problem

Faults in electronic circuits are modeled using fault models, of which the Stuck-At fault model is the most commonly used. It is assumed that faults occur and lines get permanently stuck to 0 or 1.

There are two basic principles in testing: Controllability and Observability. In a combined circuit there are also several nodes, which can all have a Stuck-At fault (for simplicity only one at a time).

Now, if you want to detect a fault in a circuit, you must be able to activate these fault and observe it at them same time (you cannot detect a fault if the circuit ends up working correctly, i.e., the fault does not propagate to the output). For this purpose, a suitable test pattern must be generated.

In the D-algorithm (Roth, 1966) one tries to make the error observable at the output by suitable assignment at the input, and not to overwrite the error.
